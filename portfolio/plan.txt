# Immersive Forest Portfolio

## Summary
Build a greenfield portfolio in `Next.js` with `React Three Fiber` as a desktop-first, cinematic 3D experience that still stays recruiter-usable.

The experience will have:
- A landing page with a forest scene and 3 visible paths: `Projects`, `Skills & Achievements`, and `Education & Co-curricular`.
- Hover interaction on each path that brightens the trail, reveals the label, and subtly animates the surrounding environment.
- A fixed ~3 second click transition that “zooms through” the chosen path and then lands inside a dedicated 3D world.
- Three destination scenes:
  - `Projects` -> waterfall environment
  - `Skills & Achievements` -> grasslands environment
  - `Education & Co-curricular` -> cave environment
- Each destination scene reveals 3-4 interactive 3D objects tied to your real content.

## Key Changes
- App structure:
  - Use routes for `/`, `/projects`, `/skills-achievements`, and `/education-cocurricular`.
  - Keep the landing page as the single entry point; destination routes are deep-linkable for recruiter convenience.
- Landing interaction:
  - Render one forest hub scene with 3 clearly separated paths.
  - On hover, each path gets a highlight treatment: emissive glow on trail markers, stronger volumetric light, slight camera bias toward that path, and visible text label.
  - On click, lock input, play a scripted camera dolly for ~3 seconds, fade/crossblend the forest into the target world, then complete navigation.
  - Begin preloading the hovered scene’s assets before click to reduce transition hitching.
- Destination scenes:
  - `Projects / Waterfall`: 3-4 project objects such as floating stone tablets, wooden signposts, or relics near the falls; clicking one opens a detail panel with project summary, stack, outcome, GitHub/live links.
  - `Skills & Achievements / Grasslands`: 3-4 objects such as totems, banners, or wind-activated structures; each object maps to grouped skills or a notable achievement.
  - `Education & Co-curricular / Cave`: 3-4 objects such as glowing crystals, carvings, or lantern altars; each reveals education milestones, clubs, leadership, competitions, or extracurricular highlights.
  - Every scene should include a persistent minimal UI with `Back to forest`, `Quick nav`, `Resume`, and `Contact`.
- Content system:
  - Store all portfolio content in structured local data, not hardcoded in scene components.
  - Use a typed content module such as `content/portfolio.ts` or JSON + TS types.
  - Scene components read from content config and generate interactive objects from that data.
- Visual/technical direction:
  - Target cinematic-stylized visuals, not full photoreal custom production.
  - Use curated GLB/environment assets plus custom lighting, fog, particles, and postprocessing.
  - No autoplay audio by default.
  - Desktop-first delivery: phones get a simplified fallback page with static visual previews and the same content in conventional cards rather than full 3D scenes.

## Public Interfaces / Types
Define a small typed content contract so the portfolio stays easy to update:
- `PortfolioPath`
  - `slug`
  - `title`
  - `sceneTheme`
  - `summary`
  - `objects`
- `PortfolioObject`
  - `id`
  - `name`
  - `kind`
  - `position`
  - `modelRef`
  - `accentColor`
  - `panel`
- `ObjectPanelContent`
  - `headline`
  - `description`
  - `metadata`
  - `links`
  - `media`
- `SceneConfig`
  - `environment`
  - `lighting`
  - `cameraTarget`
  - `transitionPreset`

Implementation should treat scene object placement as data-driven so content changes do not require scene rewrites.

## Test Plan
- Landing page shows exactly 3 interactive paths with correct labels and hover highlight states.
- Clicking each path plays one transition sequence lasting roughly 2.8-3.2 seconds and lands on the correct route.
- Hovered path starts preloading the destination scene; navigation should not visibly stall after the transition.
- Each destination scene renders 3-4 mapped objects from content data and opens the correct detail panel on interaction.
- Persistent UI works in every scene: back navigation, quick nav, resume link, and contact CTA.
- Deep links to `/projects`, `/skills-achievements`, and `/education-cocurricular` work without visiting the landing page first.
- Mobile devices receive the simplified non-3D fallback rather than a broken or low-FPS full scene.
- Reduced-motion users should get shortened or softened transition behavior on desktop.
- Validate in current desktop Chrome and Safari at minimum.

## Assumptions And Defaults
- Greenfield project with no existing design system or code constraints.
- Primary audience is recruiters and collaborators, so immersion must not block fast scanning of your work.
- Asset strategy uses prebuilt 3D kits/models with consistent art direction rather than custom modeling everything from scratch.
- `Achievements` is grouped with `Skills`, and `Co-curricular` is grouped with `Education`, to preserve the 3-path concept.
- The first implementation should use placeholder portfolio content until your final text, links, and media are filled in.
